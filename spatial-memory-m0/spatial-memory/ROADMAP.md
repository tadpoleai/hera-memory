# 空间记忆框架 — 逐步填充技术路径 (M0 → M4)

## 总原则:接口冻结,实现替换

本框架的探索方法论只有一条:**每个模块由一个冻结的接口 + 一个可替换的实现组成**。
每个里程碑做的事情,就是把某几个 M0 的"玩具实现"替换为生产实现,而接口与上层代码不动。
这保证团队可以并行探索、随时回退、每一步都有可运行的系统做对照。

```
                 ┌─────────────────────────────────────────────┐
   接口(冻结)     │ EntityStore  EventLog  VectorIndex  BlobStore│
                 │ Consolidator.ingest()   SpatialQuery.*       │
                 │ ObservationEvent (机器人端唯一上报格式)        │
                 └─────────────────────────────────────────────┘
                        ▲               ▲               ▲
   实现(替换)      M0 SQLite/numpy  M2 PG/pgvector   M3 Kafka/Redis
```

## 当前状态:M0(本仓库)

| 模块 | M0 实现 | 状态 |
|---|---|---|
| schema.py | dataclass 契约,proto 字段号已预留 | ✅ 可运行 |
| store.py | SQLite(实体+事件)+ numpy 暴力向量检索 + 本地 blob | ✅ 可运行 |
| consolidator.py | 贪心匹配 + 五种写入决策 + 负观测衰减 + 人工修正 pin | ✅ 可运行 |
| query.py | 结构化 / 语义 / 时序 / 同步 四类原语 | ✅ 可运行 |
| synthetic.py | 合成世界 + 模拟机器人 + trigram 玩具嵌入 | ✅ 可运行(M1 起逐步退役) |

M0 已验证的完整生命周期:`ADD → CONFIRM → UPDATE → DECAY → RETIRE → HUMAN_CORRECT → 增量同步`。

**M0 的已知刻意简化(即探索课题清单)**:
1. 可见性模型 = "同房间 + 半径"(无真实视锥、无遮挡剔除)
2. 跨房间移动无法重识别(旧实例 RETIRE + 新实例 ADD,而非 UPDATE)
3. 玩具嵌入无真实语义(中文查询召回差)
4. 单机器人、单进程、无并发冲突
5. 无 L0 几何数据(mesh/TSDF 为空引用)

## M1:真实感知接入 + 契约固化(预计 4–8 周)

目标:**让真实相机数据流过同一条管线**,并冻结数据契约。

- [ ] **schema → Protobuf**:按注释中的字段号迁移,codegen Python/C++ 两端;
      建立 schema 评审与版本规范(这是最难返工的部分,优先锁死)
- [ ] **真实嵌入**:`toy_embed` → CLIP/SigLIP 文本-图像双塔;
      实体增加 `embedding_model` 字段的强校验(不同模型的向量禁止混入同一索引)
- [ ] **离线首建管线 v1**(替代 SyntheticWorld):
      rosbag/mcap → 现成 SLAM(nvblox / FAST-LIVO2)→ 子图划分
      → SAM2+CLIP 开放词汇实例提取 → ConceptGraphs 式跨帧融合
      → 房间分割(先允许人工画多边形,自动化后置)→ 入库
- [ ] **人工校验 Web 工具 v0**:实例列表 + 合并/删除/改名,产出 HUMAN_CORRECT 事件
- [ ] **验收标准**:一个真实楼层入库;50 条人工标注查询的召回率 ≥ 80%;
      SyntheticWorld 保留为 CI 回归测试

## M2:整合器进化 + 存储生产化(预计 8–12 周,核心自研期)

目标:**把写路径做成技术壁垒**。这是最值得投入的阶段。

- [ ] **匹配算法升级**(consolidator._match 单点替换):
      贪心 → 匈牙利分配;加入几何形状相似度;跨房间/跨子图重识别
      (基于嵌入的全局检索 + 时空先验:"消失的杯子出现在隔壁,大概率是同一个")
- [ ] **真实视锥可见性**:相机内参 + 位姿 → 视锥体;用 L0 mesh 做遮挡剔除,
      负观测从"同房间半径"升级为"几何精确的应见未见"
- [ ] **存储替换**:SqliteStore → PostgreSQL + PostGIS(GiST 空间索引)+ pgvector(HNSW);
      NumpyVectorIndex 保留作召回率对照基线;LocalBlobStore → MinIO
- [ ] **L0 数据接入**:子图 TSDF/mesh 入 MinIO,GeometryRef 生效
- [ ] **内部基准 v1**:变化检测 P/R、匹配准确率、查询延迟 P99,接入 CI
- [ ] **验收标准**:物体移动/消失/新增的检测 F1 ≥ 0.9(自建测试集);
      跨房间重识别准确率 ≥ 70%

## M3:多机 + 时序 + 人机协作闭环(预计 12–16 周)

目标:**从单机记忆变成共享记忆**。

- [ ] **事件总线**:EventLog 写路径前置 Kafka;30 天外事件压 Parquet 冷备;
      changes_since 升级为 (submap_id, cursor) 游标协议,支持断线续传
- [ ] **多机冲突消解**:version 乐观锁 + 整合器仲裁
      (人工修正 > 高置信近期观测 > 旧观测),并发写入压测
- [ ] **工作记忆层**:Redis 缓存活跃 place 的实体集 + pub/sub 变更推送;
      SubscribeRegion 原语上线(机器人订阅所在区域的实时变更)
- [ ] **时序切片**:where_was 从全量回放升级为 周级物化切片 + 增量回放
- [ ] **人机协作 v1**:自然语言修正入口(LLM 解析 → HUMAN_CORRECT 事件);
      高风险更新(STATIC 物体被 RETIRE)进人工复核队列
- [ ] **验收标准**:3+ 机器人并发运行一周,记忆一致性无人工干预;
      端到端"人说一句话修正地图"演示

## M4:记忆智能化(持续探索期)

目标:**从数据系统变成记忆系统**,与 LLM memory 前沿持续对齐。

- [ ] **记忆管理器**(Letta/MemGPT 式):按任务预取相关区域记忆到 agent 上下文,
      暴露 query_region / update_object 等记忆工具给 LLM agent 主动调用
- [ ] **学习型衰减与遗忘**:decay_step 从手工配置 → 按物体类别/区域从历史数据学习
- [ ] **主动探索调度**:置信度地图驱动巡逻路线("最久未确认的区域优先")
- [ ] **场景图推理服务**:LLM 在 place graph + 实体上做任务落地(SayPlan/GraphEQA 路线)
- [ ] **隐式记忆插槽**:预留世界模型 latent 作为并行表示,与显式场景图互补

## 并行探索的组织建议

三条线可以同时推进,交汇点是冻结的接口:

- **感知线**(M1 主力):SLAM、开放词汇分割、实例融合 —— 产出 ObservationEvent
- **系统线**(M2/M3 主力):存储、事件总线、同步协议 —— 实现四个 Store 接口
- **算法线**(M2/M4 主力):整合器匹配、衰减策略、记忆管理 —— 迭代 Consolidator

每条线的每次改动,都必须通过 `demo.py`(合成回归)+ 内部基准(真实数据)双重验收。
